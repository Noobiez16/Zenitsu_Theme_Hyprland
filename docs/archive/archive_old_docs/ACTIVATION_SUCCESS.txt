For the complete experience with electrical "zap" sounds on clicks:

STEP 1: Install Sox (Sound Generation Tool)
────────────────────────────────────────────────────────────────────
  $ sudo pacman -S sox

STEP 2: Re-run the Electrical Icons Script
────────────────────────────────────────────────────────────────────
  $ cd ~/Project_Zenitsu_Theme
  $ ./electrify_icons_and_sounds.sh

  This will regenerate the script and create actual spark sounds
  using sox.

STEP 3: Test the Spark Sound
────────────────────────────────────────────────────────────────────
  $ ~/Project_Zenitsu_Theme/sounds/play_spark.sh

  You should hear a quick electrical "zap" sound!

WHY SOUNDS DIDN'T GENERATE:
  Sox was not installed when you ran the script, so sound files
  weren't created. The visual effects still work perfectly!

ALTERNATIVE: Use Your Own Sounds
────────────────────────────────────────────────────────────────────
  Place any .wav file at:
    ~/Project_Zenitsu_Theme/sounds/electric_spark.wav
  
  The play_spark.sh script will use it automatically.

IF ICONS LOOK WRONG OR MISSING:
  • Some icons may not render if fonts are missing
  • Install all Nerd Fonts: sudo pacman -S ttf-nerd-fonts-symbols-mono
  • Reload Waybar: pkill waybar && waybar &

IF CLICK ANIMATION DOESN'T SHOW:
  • The animation is very fast (150ms)
  • Try clicking and holding briefly
  • Make sure Thunder Glow style is active

IF HOVER EFFECTS DON'T WORK:
  • Verify Thunder Glow is active:
    $ readlink ~/.config/waybar/style.css
  • Should point to Thunder Glow.css

IF WAYBAR CRASHES:
  • Check errors: waybar 2>&1 | grep error
  • Switch to Thunder Breathing: activate_thunder_theme.sh
  • Report error to check logs
