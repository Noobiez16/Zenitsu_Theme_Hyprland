━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ISSUE:
  Thunder Glow style caused Waybar to disappear (same as before).

ROOT CAUSE:
  Complex @keyframes animations with multi-line box-shadow properties
  caused CSS parsing errors. The Waybar CSS parser couldn't handle:
    • Multiple box-shadow values across lines in keyframes
    • Complex percentage-based keyframe definitions (0%, 50%, 100%)

SOLUTION:
  1. Simplified all @keyframes to use from/to syntax instead of percentages
  2. Reduced box-shadow to single-line declarations
  3. Changed animation direction to use "alternate" instead of complex loops
  4. Kept all visual effects while ensuring CSS compatibility

  • Plays animation forward, then backward
  • Creates smooth pulse effect automatically
  • More efficient than defining reverse keyframes
  • Equivalent to 0%→100%→0% but simpler

  ✓ No noticeable performance impact
  ✓ Smooth 60fps rendering
